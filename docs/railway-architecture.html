Deployment steps on Railway
1
Connect GitHub repo
Create a new Railway project and link your GitHub repo. Railway auto-detects pushes to master.
2
Add 3 services
Add API (root: backend/src/PRDF.Lms.Api), client-ui, and admin-ui as separate Railway services from the same repo.
3
Set env variables
Paste Supabase keys into each service's Variables tab. Set VITE_API_BASE_URL on both UIs to the API's Railway URL.
4
Configure static sites
For client-ui and admin-ui set build command npm run build and publish directory dist. Railway serves static files automatically.
Important: The two Vite frontends are static sites — Railway builds them once (npm run build) and serves the /dist folder via CDN. They are not Node servers. The only live server is the .NET API. Supabase is fully managed outside Railway — you just point both UIs and the API at your existing Supabase project URL.
